Most agent tools hide the work.
We show it.

Spinner, "thinking…", a clean final diff. The industry hides the guts for demo-comfort. We bet the opposite: the process is the product.

Everywhere else

Trust me, look away
  • ✕ A spinner and a final diff. The reasoning trail is collapsed, truncated, or gone.
  • ✕ Tool calls run off-screen. You find out what happened after it's done — if you can.
  • ✕ You steer by re-prompting a black box and hoping the next attempt is better.
  • ✕ When your machine sleeps, the agent dies. Hosted, single session, closed.

BawtHub

Watch, and correct
  • ✓ Every tool call is a card with its result — command, args, output, live. Nothing collapsed by default.
  • ✓ Latency, token cost, model, +/− diff on the same row. The receipts, not a summary.
  • ✓ You catch the wrong turn while it's happening and correct it — steering by watching.
  • ✓ It's your box. Self-hosted, multi-agent, and it keeps working while you're away.

The mental model

A remote dev laptop, with agents at the keyboard.

A developer's laptop — except remote, always-on, and agent-operated.

It runs 24/7.

Close your laptop; the agents keep going. Competitors that live in your editor die when your machine sleeps.

One shared filesystem.

A single workspace across bridges — a team, not one task-runner in a sandbox. One agent's commit is the next one's start.

Origin

It started as a CLI.

From a one-line shell command to a team of agents you watch build software. Same core API the whole way.

v0 · CLI

One command, one bot.

llm -b loopy "what's up" — local, file-based history. Named bots with their own prompts, from day one.

v1 · llm-bawt server

One API, every model.

Every provider — Claude, OpenAI, Grok, local GGUF, Ollama, vLLM — behind one OpenAI-compatible API. Memory into Postgres + pgvector.

v2 · unmute → BawtHub

Voice. Then a real UI.

Born as unmute, a realtime voice framework — STT, TTS, hotword. The web UI grew around it; voice never went away.

v3 · agent bridges

Agents in the same chat.

OpenClaw, then the Claude Code SDK, then Codex — one bridge pattern. The hard problem: streaming tool calls + results into one UI.

v4 · task system

Projects, tasks, sign-off.

Plan in markdown, dispatch to a bot, review. Agents manage their own tasks via MCP — and ping each other.

v5 · observable agentic development

Make the work the surface.

The Relay: live turn metadata, +/− diff chips, nested sub-agent tools, streamed approvals. The crown jewel — you watch the agents work.

Still private — and honest about it.

A single-user, self-hosted system under active development. The code is on GitHub; the running instance is Nick's own box, building itself. Not open to the public yet — on purpose.
