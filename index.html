The work is usually hidden.
Here it's the point.

Agent harnesses have world-class guts — but you usually watch them through a collapsed chat you babysit with a console open, or a --verbose flood you can't scroll back through. BawtHub takes those same guts and renders them so you can actually sit and watch: the process is the product.

The usual surface

Great guts, hard to watch
  • ✕ Tool calls flash by inline or stay collapsed in an embedded terminal. To really see what happened, you keep a separate console open.
  • ✕ Run more than one agent and they share one terminal — output interleaves into noise.
  • ✕ You toggle --verbose to see the detail, it floods the screen for a single command, and scrolling back later is painful.
  • ✕ Session-bound: end it or sleep the machine and the trail is gone.

BawtHub

Same guts, made watchable
  • ✓ Built on the same real harnesses (Claude Code SDK, Codex) — their streams render as tool-type cards, with the chat as the focal point. Nothing collapsed by default.
  • ✓ Latency, token cost, model, +/− diff on the same row. The receipts, not a summary — and they're still there to scroll back through later.
  • ✓ Many agents, one board — each turn in its own lane instead of sharing a terminal.
  • ✓ It's your box. Self-hosted, multi-agent, and it keeps working while you're away.

Give each agent its own personality, model, voice, and specialty — then edit its behavior on the fly. One filesystem, many agents, working in parallel while you watch the whole board.

The mental model

A remote dev laptop, with agents at the keyboard.

A developer's laptop — except remote, always-on, and agent-operated.

It runs 24/7.

Close your laptop and the agents keep going. It lives on your box, not in your editor — nothing dies when your machine sleeps.

One shared filesystem.

A single workspace across bridges — a team, not one task-runner in a sandbox. One agent's commit is the next one's start.

You control it by chatting.

Point it at any project. Steer in plain language and watch the work land.

What they build runs on the host.

BawtHub is the cockpit, not your prod host. Agents push to your repo; your real deploy is your own CI.

Not just a desktop app

Close the laptop. Keep shipping — from your phone.

Same app, same guts, fully responsive. Start on desktop, close it, go walk the dog — then check progress, read tool calls, and push fixes from the device in your pocket. Not a stripped-down companion view; the whole rich environment. All real captures.

One surface, every harness.

You're not bound to one provider's app — juggling Codex in one window, Claude in another, a terminal, an editor plugin, each with its own workflow. Every model and every harness runs in the same chat. When one provider's usage window runs dry, you swap the model behind the conversation and keep going — your development experience never changes underneath you.

Local models get a first-class seat too — the provider desktop apps won't let you bring your own. A smaller crowd today, but as local models get better, running everything in one consistent place is where it pays off.

Origin

It started as a CLI.

From a one-line shell command to a team of agents you watch build software. Same core API the whole way.

v0 · CLI

One command, one bot.

llm -b loopy "what's up" — local, file-based history. Named bots with their own prompts, from day one.

v1 · llm-bawt server

One API, every model.

Every provider — Claude, OpenAI, Grok, local GGUF, Ollama, vLLM — behind one OpenAI-compatible API. Memory into Postgres + pgvector.

v2 · unmute → BawtHub

Voice. Then a real UI.

Born as unmute, a realtime voice framework — STT, TTS, hotword. The web UI grew around it; voice never went away.

v3 · agent bridges

Agents in the same chat.

OpenClaw, then the Claude Code SDK, then Codex — one bridge pattern. The hard problem: streaming tool calls + results into one UI.

v4 · task system

Projects, tasks, sign-off.

Plan in markdown, dispatch to a bot, review. Agents manage their own tasks via MCP — and ping each other.

v5 · observable agentic development

Make the work the surface.

The Relay: live turn metadata, +/− diff chips, nested sub-agent tools, streamed approvals. The crown jewel — you watch the agents work.

Open backend, private everything else.

The llm-bawt backend is open on GitHub. The BawtHub frontend and the running instance stay private — a single-user, self-hosted system that wrote this line too. Under active development, on purpose.
